# منصّة فِكرة — نظام إدارة مهام التسويق (Laravel)

نظام عربي (RTL) لإدارة سير عمل فريق تسويق من ٤ أفراد: من الفكرة ← الخطة ← التصميم ← التنفيذ ← النشر ← الصيانة، مع محرّك نقاط/تارجت ومحرّك رواتب سرّي وصلاحيات حسب الدور. مبني على **Laravel 11 + MySQL + Blade + Tailwind (RTL) + spatie/laravel-permission**.

> هذا الكود يحتوي كامل منطق التطبيق (النماذج، الخدمات، المتحكمات، الترحيلات، البذور، الواجهات). يبقى عليك فقط جلب اعتماديات Laravel عبر `composer install` على جهازك.

---

## التشغيل خطوة بخطوة

```bash
# 1) جلب الاعتماديات (يتطلب اتصالاً بالإنترنت + PHP 8.2+ و Composer)
composer install

# 2) إعداد البيئة
cp .env.example .env
php artisan key:generate

# 3) اضبط قاعدة بيانات MySQL في ملف .env
#    DB_DATABASE=fikra  /  DB_USERNAME  /  DB_PASSWORD
#    ثم أنشئ القاعدة:
#    mysql -u root -p -e "CREATE DATABASE fikra CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 4) الترحيلات + البيانات التجريبية (مستخدمون + شهران من البيانات)
php artisan migrate --seed

# 5) رابط التخزين العام (لرفع التصاميم)
php artisan storage:link

# 6) التشغيل
php artisan serve
# افتح: http://localhost:8000
```

> ملاحظة: إعدادات `.env` الافتراضية تستخدم `SESSION_DRIVER=file` و`CACHE_STORE=file` حتى يعمل النظام دون جداول إضافية. يمكنك تغييرها لاحقاً.

## الوحدات التسع

1. **لوحة التحكم** — ملخّص بفلتر فترة (من/إلى) وحلقة تقدّم نحو التارجت.
2. **إدارة المستخدمين** (أدمن) — إضافة/تعديل/تعطيل وإعادة كلمة المرور.
3. **خطة المحتوى** — جدول شهري كامل مع اعتماد/مراجعة/عدم موافقة ورفع تصميم المصمم.
4. **إدارة المهام** — لوحة كانبان بسير العمل الموحّد.
5. **صيانة الموقع** (أدمن/مشرف) — عمليات تُحتسب نقاطها لمشرف المحتوى.
6. **النقاط والتارجت** — بطاقات الفريق ومعادلة المشرف (+30% من المصممين).
7. **الحضور والإجازات** — طلبات وموافقات ورصيد شهري.
8. **الرواتب** (أدمن فقط، سرّي) — مكافآت وخصومات وصافي لكل شهر.
9. **التقارير** (أدمن/مشرف) — تبويبات قابلة للتصدير CSV (بترميز UTF‑8، بلا بيانات رواتب).

## فصل الشهور (مهم)

كل شهر **مستقلّ تماماً**: الخطة والمهام والنقاط والتارجت والرواتب والإجازات والتقارير كلها مرتبطة بالشهر النشط. يُحفظ الشهر النشط في الجلسة، ويُبدّل من منتقل الشهر أعلى الشاشة (سهم سابق/تالي + شبكة شهور السنة). الاشتقاق يتم من تاريخ كل عنصر عبر `scopeForMonth()` في النماذج.

## منطق النقاط (`app/Services/PointsService.php`)

- لكل نوع مهمة قيمة ثابتة (`app/Support/WorkTypes.php` هو المصدر الموحّد).
- ملف المشروع الإبداعي/السريع: نقاط إضافية. التأخر في التسليم: نصف النقاط.
- «فكرة ريلز جديدة» تُحتسب فقط عند وصولها لمرحلة «منشور».
- مشرف المحتوى: نقاطه المباشرة + 30% من إجمالي نقاط المصممَين.

## منطق الرواتب (`app/Services/PayrollService.php`)

- مكافأة 5% عند تجاوز التارجت.
- خصم عدم تحقيق التارجت بحد أقصى 20% من الراتب.
- الإجازات الزائدة عن يومين: الأجر اليومي (الراتب/30) عن كل يوم زائد.

## الصلاحيات والسرّية

- الأدوار عبر spatie؛ الوسطاء `role:` تحمي المسارات.
- الراتب في `$hidden` بالنموذج + بوابة `view-salaries` (أدمن فقط) + حماية على مستوى المتحكم + استثناؤه من كل التقارير والتصدير.

## بنية المشروع (أهم الملفات)

```
app/
  Support/WorkTypes.php            ← المصدر الموحّد لأنواع العمل والنقاط والقوائم
  Models/                          ← User, Task, ContentPlan, MaintenanceItem, Leave (مع scopeForMonth)
  Services/PointsService.php       ← محرك النقاط (شهري + نطاق تاريخي للوحة التحكم)
  Services/PayrollService.php      ← محرك الرواتب
  Providers/AppServiceProvider.php ← بوابتا view-salaries و manage-users
  Http/Controllers/               ← 9 وحدات + Auth
  Http/Requests/                  ← تحقّق المدخلات
database/migrations/              ← الجداول (مستخدمون موسّعون + spatie + خطة/مهام/صيانة/إجازات)
database/seeders/                 ← أدوار + ٤ مستخدمين + بيانات شهرين
resources/views/                  ← Blade (RTL، الشريط الجانبي يميناً، منتقل الشهر)
routes/web.php                    ← المسارات مع حماية الأدوار
```

## ملاحظات

- القوائم المرجعية (المنصات، أنواع المحتوى/المنشور، الحالات) معرّفة مرّة واحدة في `WorkTypes`. يمكن ترقيتها إلى جداول lookup إذا رغبت بإدارتها من قاعدة البيانات.
- التصميم يستعمل Tailwind وAlpineJS عبر CDN ليعمل فوراً دون بناء أصول. للإنتاج يُنصح بتثبيتهما عبر Vite.
- نموذج HTML تفاعلي كامل (واجهة مرجعية بصرية) متاح بشكل منفصل ويُطابق هذا الكود.

## تحديث الإصدار الثاني (الصلاحيات + الميزات)

بعد سحب هذا الإصدار فوق مشروعك:

```bash
php artisan migrate                 # جداول/أعمدة/فهارس جديدة (أنواع المهام، حقول المحتوى، الوصف)
php artisan db:seed --class="Database\Seeders\RolePermissionSeeder"
php artisan db:seed --class="Database\Seeders\TaskTypeSeeder"
php artisan storage:link            # للمرفقات المرجعية وتصاميم المصممين
php artisan optimize:clear
```

تشغيل الاختبارات الآلية:

```bash
php artisan test
```

أهم ما أُضيف: عزل البيانات لكل مستخدم وفرضها على الخادم (سياسات)، واجهة إدارة الأدوار والصلاحيات، إدارة أنواع المهام، لوحة كانبان بالسحب والإفلات، حقول محتوى جديدة (نص المنشور/ملاحظات/مرجع بصري)، اعتماد متعدّد بأربع حالات، نماذج تعديل مضمّنة، فهارس أداء، واختبارات.
